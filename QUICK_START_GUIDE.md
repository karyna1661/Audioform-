# 🚀 Audioform Quick Start Guide

## For New Users (5-Minute Setup)

### Step 1: Sign In (30 seconds)
```
1. Go to: https://your-railway-url.com
2. Click "Get Started"
3. Sign in with Google
4. ✅ You're in!
```

---

### Step 2: Create Your First Survey (3 minutes)

**Option A: Use Template (Fastest)**
```
1. Click "Create new survey"
2. Scroll to "Pre-built Templates"
3. Click "Experience → Friction → Desire"
4. Done! Questions auto-filled.
```

**Option B: Custom Questions**
```
1. Click "Create new survey"
2. Fill in decision context:
   - What are you changing? → "Onboarding flow"
   - What kind of change? → "Simplify"
   - What outcome? → "Conversion"
3. Browse categories, click to add questions
4. Drag to reorder
5. Edit any question to customize
```

---

### Step 3: Publish (1 minute)
```
1. Review quality scores (aim for 80%+)
2. Click "Publish" button
3. ✅ Survey link copied to clipboard!
```

---

### Step 4: Share & Collect (ongoing)
```
Share your link:
- Email to users
- Slack/Discord
- Twitter/LinkedIn
- Embed on website

Watch responses roll in! 📬
```

---

### Step 5: Review & Decide (15 minutes)
```
1. Get email notification
2. Go to /admin/responses
3. Listen to responses (play at 1.5x speed)
4. Read transcripts
5. Bookmark key insights ⭐
6. Identify patterns (look for repeated themes)
7. Make your decision!
```

## Pro Tips

### Question Quality Hacks
✅ **Good**: "What confused you the most when you first used this product?"
❌ **Bad**: "Did you like our product?" (yes/no question)

**Use these triggers:**
- "Tell me about the last time..."
- "What was the moment when..."
- "Walk me through how you..."
- "Describe a specific example of..."

### Best Practices
- **1-3 questions max** (voice surveys work best short)
- **Ask for stories, not opinions**
- **Include timeframe** ("last week", "first time")
- **Target 80%+ quality score** before publishing
- **Listen at 1.5x-2x speed** for faster review

### Response Collection Strategy
```
Week 1: Send to 5-10 power users
Week 2: Analyze patterns, iterate survey
Week 3: Send to broader audience (50+ users)
Week 4: Make data-driven decision
```

## Common Use Cases

### 1. Onboarding Optimization
```
Template: "Onboarding Insights"
Questions:
- "What confused you the most when you first used this product?"
- "At what moment did you almost give up?"
- "What would have made onboarding feel effortless?"

Decision: Simplify step 2, remove unnecessary fields
```

### 2. Feature Validation
```
Template: "Feature Validation"
Questions:
- "How are you currently solving this problem today?"
- "What feature would make this product indispensable?"
- "What would convince you to adopt this feature immediately?"

Decision: Build feature X, prioritize for Q2
```

### 3. Pricing Page Testing
```
Template: "User Feedback"
Questions:
- "What was your first reaction when you saw the pricing?"
- "What felt unclear or confusing about the plans?"
- "What would make you more confident in choosing a plan?"

Decision: Add annual discount, clarify enterprise tier
```

### 4. PMF Discovery
```
Template: "PMF Discovery"
Questions:
- "What problem were you hoping this product would solve for you?"
- "What part of the experience felt confusing or frustrating?"
- "What would make this product something you use every week?"

Decision: Pivot to serve [specific use case] better
```

## Troubleshooting

### "No responses yet"
✅ Normal for new surveys. Share your link more widely.

### "Quality score too low (<60%)"
- Add more context to question
- Include specific timeframe
- Avoid yes/no framing
- Use "what/how" instead of "did you"

### "Microphone not working"
- Check browser permissions (camera/mic)
- Try Chrome/Edge (best support)
- Refresh page and try again

### "Transcription has errors"
- Usually 95%+ accurate
- Manual edit if critical words wrong
- Speak clearly for better accuracy

### Common Questions

**Q: How long should responses be?**
A: 20-60 seconds is ideal. Shorter = surface level. Longer = rambling.

**Q: Can I export responses?**
A: Yes! Click export button to download audio + transcript as ZIP.

**Q: Can I share with my team?**
A: Currently single-user. Team features coming soon.

**Q: How many questions should I ask?**
A: 1-3 max. Voice takes effort. Respect your users' time.

**Q: What if no one responds?**
A: 
- Ask fewer questions (1 is enough!)
- Offer incentive (coffee chat, discount)
- Make it about THEM, not you ("Help us improve" vs "Give feedback")
