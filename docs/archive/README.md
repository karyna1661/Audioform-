# Archive

This folder holds historical documentation that is still useful for internal reference but should not appear as part of the main public GitHub surface.

## Contents

- `releases/` - launch checklists, rollout status, and dated release summaries
- `incidents/` - bug investigations, fix summaries, and troubleshooting notes
- `ops/` - deployment troubleshooting, environment notes, and one-off operator guides
- `data/` - migration notes and existing-data handling guidance
- `product/` - archived product snapshots and dated reference material

## Naming Convention

- `release-*` for dated launch and rollout material
- `incident-*` for bug investigations and fix writeups
- `ops-*` for operator notes and environment troubleshooting
- `data-*` for migration and data-handling notes
- `product-*` for archived product snapshots

Prefer `README.md`, `docs/README.md`, `docs/product/`, and `docs/internal/` for current documentation.
