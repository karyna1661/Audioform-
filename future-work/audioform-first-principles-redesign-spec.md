# Audioform First-Principles Redesign Spec

Last updated: 2026-03-04

## Core Job
Builders do not need more feedback volume. They need decision-ready signal.

## Product Promise
Ask one focused question, collect conviction-rich voice responses, and decide what to ship next.

## Non-Negotiable Product Rules
1. Every survey is anchored to one explicit product decision.
2. Intent mode must act as a lens on that exact decision.
3. Prompt suggestions must be generated from structured decision inputs.
4. Respondents must only answer published builder-authored prompts.
5. Readiness should penalize vague decisions before publish.

## Builder Flow (Simplified)
1. Define the decision:
- Change target
- Change type
- Desired outcome
- Optional detail override
2. Choose intent lens:
- Validate Direction
- Find Weak Spots
- Find Confusion
- Capture Emotion
3. Compose and publish:
- Apply decision-aligned starter prompts
- Edit for specificity
- Publish and share

## UX Copy Principles
1. Use decision language, not survey language.
2. Replace generic labels with causal labels ("truth lens", "decision anchor").
3. Explain each section by what decision clarity it contributes.
4. Keep one primary CTA per stage.

## Readiness Model
Required for strong readiness:
1. Title present
2. Change target selected
3. Change type selected
4. Desired outcome selected
5. Decision sentence specific (not vague)
6. At least 2 prompts
7. At least 1 depth prompt

## Success Criteria
1. Builders can explain their next product decision after first 5 responses.
2. Shared survey prompts always match authored published prompts.
3. Prompt suggestions vary meaningfully by decision + intent combination.
4. Dashboard replay supports play/pause and clear response review.
